Modelo via def Python

Todo o provider do LLM fica aqui: URL, modelo, autenticação, multimodalidade, timeouts e parâmetros.

def única
Defina gerar_resposta(messages, cfg) ou executar(messages, cfg). O Maestro sempre envia a lista messages e espera texto de volta.

Assistente configurável

Defina a interface HTML e o Python que recebe o submit. Assim o chat pode virar texto, texto+imagem, áudio ou qualquer outro formulário.

UI + def
A rota /chat renderiza este HTML no GET. No POST, executa atender(request, cfg, helpers) ou executar(...); se vazio, usa o chat de texto simples.

Operação local

Configurações do runtime do agente, separadas do modelo.

agente
Fluxo mínimo agora é configurado: cole uma def em "Dispatch da agenda" e outra em "Core do Maestro". Se uma delas ficar vazia, o agente registra o problema em vez de usar um core padrão escondido.
Core avançado em Python
Cole aqui os cores em Python. Não há Maestro/agenda padrão escondidos: se ficar vazio, o agente não executa aquele fluxo.
