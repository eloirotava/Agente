Hooks de entrada

Receba JSON externo em /hook/{slug}, responda rápido ao chamador e processe a mensagem resultante no Maestro em background.

HTTP JSON
Novo hook
Autenticação, validação de token e regras específicas ficam dentro do próprio Python do hook. A descrição cadastrada também é enviada como contexto ao Maestro.
Exemplo: hook default para a agenda

Crie um hook ativo com slug entrada_agenda e coloque esse mesmo slug em Configurações → Hook default da agenda.

def receber(payload: dict, headers: dict):
    return {
        "origem": payload.get("origem", "AGENDA"),
        "mensagem": payload.get("prompt", ""),
        "contexto": {
            "tipo": payload.get("tipo"),
            "titulo": payload.get("titulo"),
            "horario_scan": payload.get("horario_scan"),
            "payload_bruto": payload,
        },
    }
Chamada padrão OpenAI-compatible

Para chamar o Maestro sem hook, use /api/maestro com formato de chat completions.

curl -s -X POST http://127.0.0.1:8081/api/maestro \
  -H 'Authorization: Bearer SEU_TOKEN_LOCAL_DO_MAESTRO' \
  -H 'Content-Type: application/json' \
  -d '{
    "model":"botmig-maestro",
    "origem":"cliente-openai",
    "messages":[{"role":"user","content":"oi"}]
  }'
